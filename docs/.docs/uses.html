Getting Started

NullName DB is a zero-configuration, file-based database that works immediately. No installation, no setup, no payment. Just start using it.

First Query

# Save data
https://yourdb.com/q?q=score=100

# Retrieve data
https://yourdb.com/q?q=score

# Returns: 100
Tip: You can use any HTTP client: browser, curl, Postman, or your programming language's HTTP library.

Dashboard Access

For a visual interface, access the dashboard at /dashboard. Login with your admin credentials (default: admin / nullname2025).

https://yourdb.com/dashboard

Users & Roles System

NullName DB has a complete user management system with role-based access control (RBAC). Each user has a role that determines their permissions.

User Roles

RolePermissionsDescription
ROOTAll permissions (*)Full system access, can do anything
ADMINread, write, delete, create, manage_users, manage_system, backup, restore, forceFull administrative access
EDITORread, write, update, create, backupCan create and modify data, create backups
USERread, write, updateStandard user with read/write access
VIEWERreadRead-only access

User Management Commands

# Create a new user
/q?q=signup.username.password

# Login (returns session key)
/q?q=login.username.password

# List all users (admin only)
/q?q=user.list

# Change user role (admin only)
/q?q=user.role.username.newrole

# Delete user (admin only)
/q?q=user.delete.username

# Change your own password
/q?q=user.changepass.oldpassword.newpassword

Authentication Methods

NullName DB supports multiple authentication methods:

  • Session Key - Obtained after login, use &ses=session_key in queries
  • API Key - Generate persistent keys for applications
  • Root Key - Emergency access using ROOT_KEY from .env
  • Public Access - Read operations are public by default (can be disabled)
Security Note: Change the default admin password immediately after first deployment!

Query System

NullName DB uses a simple, intuitive query language. All queries are sent via the q= parameter.

Simple Key-Value Operations

# Store a value
/q?q=score=100

# Retrieve a value
/q?q=score

# Delete a value
/q?q=score=

Structured Data Operations

# Create database
/q?q=create.mydb

# Add data
/q?q=add.mydb.users.name.John
/q?q=add.mydb.users.age.25

# Get all users
/q?q=get.mydb.users

# Get specific user by ID
/q?q=get.mydb.users.1

# Update user
/q?q=update.mydb.users.1.name=Johnny

# Delete user
/q?q=delete.mydb.users.1

Response Formats

You can specify the response format using the format parameter:

FormatParameterUse Case
JSON&format=jsonAPIs, applications (default)
Plain Text&format=textTerminals, curl, quick views
Table&format=tableHuman-readable ASCII table
CSV&format=csvExcel, data analysis
 
# Get as CSV
/q?q=get.mydb.users&format=csv

# Get as plain text
/q?q=get.mydb.users&format=text

File Management

NullName DB includes built-in file storage. Upload files via URL or direct upload.

Upload from URL

# Store file reference (URL only)
/q?q=logo=https://example.com/image.jpg

# Download and store file locally
/q?q=logo=https://example.com/image.jpg.upload

Upload via Form

# POST to /upload endpoint
curl -X POST -F "file=@/path/to/file.jpg" https://yourdb.com/upload

File Operations

# List uploaded files
/q?q=files.list

# Get file info
/q?q=files.info.filename.jpg

# Delete file
/q?q=files.delete.filename.jpg
Note: Maximum file size is 50MB by default. This can be changed in the .env configuration.

Version Control (Git-like)

NullName DB includes a complete Git-like version control system. Track changes, create branches, merge, undo, and rollback.

Commits

# Create a commit
/q?q=commit "initial setup"

# View commit history
/q?q=commits

# Checkout a specific commit
/q?q=checkout.commit_xxx

Branches

# Create a branch
/q?q=branch.feature-login

# Switch to branch
/q?q=switch.feature-login

# List all branches
/q?q=branches

# Merge branch
/q?q=merge.feature-login.into.main

Undo & Redo

# Undo last change
/q?q=undo

# Redo last undone
/q?q=redo

# Force back 1, 2, or 3 commits
/q?q=f1
/q?q=f2
/q?q=f3
Pro Tip: Use f1, f2, f3 shortcuts for emergency recovery!

Backup & Restore

Automatic and manual backup system to protect your data.

Manual Backup

# Create a backup
/q?q=backup

# Create named backup
/q?q=backup.my_backup_name

# List all backups
/q?q=backups

# Restore from backup
/q?q=restore.backup_name

Automatic Backup

Configure automatic backups in the .env file:

BACKUP_INTERVAL_HOURS=24
MAX_BACKUPS_KEEP=10
BACKUP_COMPRESSION=true
Important: Regular backups are recommended. Test your restore process periodically.

Frequently Asked Questions

What is NullName DB?

A zero-configuration, file-based database that requires no setup, no installation, and no payment. Just start using it with simple URL queries.

Is it really free?

Yes. Completely free. No premium tiers. No payment methods. No hidden costs. Forever.

Where is my data stored?

Data is stored as encrypted JSON files in the /database/path/ directory on your VPS. You own your data completely.

How do I secure my instance?

Change the default admin password immediately. Use HTTPS in production. Set up IP whitelisting. Disable public signup if not needed.

Can I use it for production?

Yes, but we recommend testing first. For large-scale production, consider adding a reverse proxy (Nginx) and load balancer.

What are the limits?

No hard limits, but performance depends on your VPS resources. The system can handle millions of records with proper hardware.

How do I update?

Run git pull in the installation directory, then restart the service with sudo systemctl restart nullname.

Can I contribute?

Yes! NullName DB is open source. Visit github.com/nullnamedb to contribute.

How do I get support?

Community support is available via GitHub Issues. For enterprise support, contact your system administrator.
